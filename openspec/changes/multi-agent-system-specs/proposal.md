## Why

企業級簡報搜尋系統長期面臨的核心痛點是 OCR 依賴性高、幻覺率高、語義理解不足。傳統方案透過文字提取進行關鍵詞匹配，無法理解視覺內容的複雜語義，導致檢索精度低於實際需求。同時，推理模型的輸出往往缺乏真實視覺證據支撐，無法在企業環境中獲得關鍵決策者的信任。

本變更引入四層多代理專業化架構，利用最新的 ColPali（視覺語言檢索）、ImageBind（多模態對齊）、MM-R5（可解釋推理）與 Argos（視覺驗證）等突破性技術，實現端到端的零幻覺、高可解釋、企業級簡報檢索體驗。

## What Changes

- **全新視覺優先檢索管道**：捨棄 OCR，直接將 PPT 頁面渲染為高畫質圖像，利用 ColPali 進行 1024 維多向量視覺特徵提取，支援 128 維密集向量與延遲交互（Late Interaction）機制
- **大規模多向量檢索系統**：整合 LanceDB 磁碟型向量資料庫，實現雙層檢索策略（粗級篩選 + MaxSim 精細比對），支援百萬級投影片語料的毫秒級查詢
- **可解釋性推理重排模組**：搭載 MM-R5 多模態推理模型，為每份檢索結果生成詳細的思維鏈推理過程（視覺感知 → 語義對齊 → 邏輯推論 → 信度評估），增強用戶信任度
- **抗幻覺代理驗證機制**：實現 Argos 框架的視覺定位驗證，檢查推理結果是否與投影片視覺內容完全對齊，自動偵測並降權缺乏證據的推論，確保 100% 的結果準確性與可審計性
- **統一多模態特徵空間**：利用 ImageBind 建立文字、圖像等異質模態的共享向量空間，支援純文字查詢、圖像查詢、混合查詢的無縫轉換

## Capabilities

### New Capabilities

- `vision-ingestion-agent`: PPT 檔案的高保真視覺特徵提取與多向量生成。包含檔案解析、圖像渲染、ColPali 視覺編碼、ImageBind 多模態對齊等子模組
- `lakehouse-retrieval-agent`: 基於 LanceDB 的大規模多向量檢索管理系統。實現向量篩選（IVF/LSH）與 MaxSim 延遲交互精細比對的雙層檢索策略
- `reasoning-reranker-agent`: 多模態思維鏈推理與結果重排。搭載 MM-R5 模型，生成每份候選投影片的邏輯推論過程與置信度評分
- `argos-verification-agent`: 視覺證據驗證與幻覺偵測。執行推論-視覺映射檢查、動態風險評分與審計追蹤，消除模型幻覺
- `multi-modal-vector-space`: 統一的多模態向量特徵空間。文字編碼器、視覺編碼器、音頻編碼器（未來擴展）均映射至同一 1024 維（或 512 維）的共享空間
- `hybrid-retrieval-pipeline`: 混合檢索管道。支援向量檢索、全文檢索（FTS）、以及兩者融合的靈活檢索模式
- `interpretable-reasoning-chain`: 結構化推理鏈生成。多步驟推理過程（Step 1: Visual Perception → Step 2: Query Understanding → ... → Step 5: Confidence Assessment），每步均可量化與可視化

## Impact

**架構變更**：從傳統的 OCR + 稀疏向量檢索 → ColPali 多向量 + 雙層檢索 + 多模態推理驗證

**資料流變更**：
- 輸入：PPT 檔案 → 結構化圖像 + 多向量表示 → 統一特徵空間
- 處理：初步檢索 → 推理重排 → 視覺驗證 → 最終排序
- 輸出：排序列表 + 推理過程 + 驗證報告 + 證據地圖

**API 變更**：引入四個新的代理微服務，每個代理暴露標準化的異步消息接口（Message Queue）與同步 REST/gRPC 端點

**依賴項**：
- Python 3.10+, PyTorch 2.0+, Transformers (HuggingFace)
- LanceDB (最新版)
- Streamlit 1.28+
- FastAPI 0.100+ (可選，用於生產部署)
- LibreOffice (無頭模式，用於 PPT 渲染)

**效能 SLA**：
- 單頁視覺特徵提取：< 2 秒
- 檢索延遲（百萬級語料）：< 200ms（端到端）
- 推理延遲（每個候選）：1-3 秒
- 驗證延遲：500-1000ms
- 系統整體精度：MRR > 0.75, NDCG@10 > 0.78, 幻覺檢測準確度 > 92%
