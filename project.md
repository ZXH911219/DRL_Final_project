# 次世代多模態 PPT 視覺與推理檢索系統

## 1. 專案名稱與簡介

### 專案名稱
**次世代多模態 PPT 視覺與推理檢索系統**（Next-Generation Multimodal Vision and Reasoning Retrieval System for Presentation Slides）

### 簡介
本系統是一個端到端（End-to-End）的企業級多模態檢索架構，基於最新學術與業界突破性技術的綜合創新。相較於傳統 OCR 依賴的簡報檢索方案，本系統直接將 PPT 頁面渲染為高畫質圖像，並利用前沿的視覺語言模型（Vision-Language Model）進行深度特徵提取。

核心創新包括：
- **無 OCR 視覺優先架構**：利用 ColPali（Contextualized Late Interaction）機制跳過文字識別步驟，直接對視覺內容進行語義檢索
- **共享向量空間**：透過 ImageBind 建立統一的多模態特徵空間，支援文字、圖像、複合查詢的無縫轉換
- **大規模多向量檢索**：藉由 LanceDB 的磁碟型向量資料庫解決 ColPali 產生的龐大多向量儲存與效能瓶頸
- **可解釋性推理重排**：採用 MM-R5 多模態重排模型，透過思維鏈（Chain-of-Thought）推理生成每筆檢索結果的詳細論證過程
- **抗幻覺驗證防線**：集成 Argos 框架的代理驗證機制，確保所有推論與排序結果均有投影片的視覺證據支撐，完全消除模型幻覺

**系統願景**：提供零幻覺、高精準、高可解釋性的企業級簡報搜尋體驗，支援複雜的多模態查詢與可審計的決策過程。

## 3. 系統架構與功能列表

### 3.1 無 OCR 視覺攝取（OCR-Free Visual Ingestion）
**目標**：完全捨棄傳統 OCR 流程，直接將 PPT 轉換為高保真圖像並進行視覺特徵提取。

**工作流程**：
1. PPT 檔案藉由 `python-pptx` 解析獲取結構化資訊（頁面數、佈局等）
2. LibreOffice / Pillow 高畫質渲染每一頁面（600 DPI），生成標準化圖像（如 1024×768）
3. ColPali 模型對每張圖像進行分片處理：
   - 將圖像分割為 1024 個固定視覺區塊（patch）
   - 每個區塊通過視覺編碼器（基於 ViT）轉換為 128 維稠密向量
   - 形成多向量表示（Multi-vector Representation）
4. ImageBind 統一層確保異質模態輸入（文字查詢、參考圖像）能映射至同一特徵空間

**產出物**：
- 序列化的多向量集合（Numpy Arrays / Parquet 格式）
- 投影片級別的元資料（頁碼、來源路徑、時間戳）
- 視覺特徵的統計摘要（向量分佈、密度指標）

---

### 3.2 兩階段高效檢索（Two-Stage Efficient Retrieval）
**目標**：解決海量多向量的計算複雜度，透過分層策略達到毫秒級檢索延遲。

**第一階段 - 向量過濾（Vector Filtering）**：
- 利用 LanceDB 的索引結構對儲存的多向量進行粗級篩選
- 可選的全文檢索（FTS）層針對提取的文件層級信息進行預過濾
- 從百萬級投影片候選中快速縮小至前 100～500 個候選

**第二階段 - MaxSim 精細比對（MaxSim Fine-grained Matching）**：
- 對縮小後的候選集合執行 ColPali 核心的延遲交互（Late Interaction）演算法
- MaxSim 運算：$\text{score} = \max_{i,j}(\text{query\_vector}_i \cdot \text{doc\_vector}_{i,j})$
- 利用向量化運算（GPU 加速）實現毫秒級重新評分
- 最終輸出排序後的投影片列表與相似度分數

**效能指標**：
- P@1 (Precision at 1): > 85%
- MRR (Mean Reciprocal Rank): > 0.75
- 端到端檢索延遲：< 200ms（含模型推理）

---

### 3.3 可解釋性推理重排（Interpretable Reasoning-based Reranking）
**目標**：透過多模態推理生成清晰的排序決策邏輯，賦予系統高度的可解釋性。

**工作流程**：
1. 接收初步檢索結果（前 10-20 個候選投影片）
2. MM-R5 模型針對每個候選生成**多步驟推理過程**：
   - **感知階段**：提取投影片中的關鍵視覺元素（標題、圖表、文字區塊位置）
   - **語義對齐**：計算使用者查詢與視覺元素間的語義相似度
   - **推理鏈**：逐步論證為何此投影片符合查詢需求，包含具體的視覺證據引用
3. 基於推理過程的置信度與完整性分配重排分數
4. 輸出排序結果 + 詳細推理說明

**可解釋性產出示例**：
```
查詢：「機器學習在金融風控的應用」
投影片 #42：
  推理過程：
  1. 視覺檢測：識別標題「AI-Driven Risk Management」✓
  2. 語義對齐：標題與查詢的語義距離：0.92（高相關）✓
  3. 內容驗證：投影片含有風控相關的數據圖表與模型架構圖 ✓
  最終評分：9.2/10 (機器學習應用層面的直接相關性 & 金融風控背景)
```

---

### 3.4 抗幻覺代理驗證（Anti-Hallucination Agentic Verification）
**目標**：作為系統輸出的最後防線，確保所有推論與排序結果均基於投影片的真實視覺證據。

**驗證機制**：
1. **推論與視覺映射檢查**：
   - 提取推理過程中提及的關鍵視覺元素（圖表位置、文字內容、視覺特徵）
   - 在原始投影片圖像中執行視覺定位（Spatial Grounding），確認這些元素真實存在
   - 計算提及度與實際覆蓋度的一致性比率

2. **幻覺偵測與降權**：
   - 若推論內容無法在投影片中找到視覺支撐（缺乏證據或虛構內容），觸發**幻覺警報**
   - 對該結果的排序權重進行動態降低，或將其標記為「低信度」
   - 可選：重新觸發檢索或詢問使用者是否需要不同的搜尋策略

3. **審計追蹤與透明度**：
   - 記錄所有驗證過程的中間結果，生成可審計的決策日誌
   - 提供視覺化的「證據地圖」，展示推論對應的投影片區域
   - 用戶可直觀查看「為什麼這個答案是可信的」

## 5. 成功指標與評估框架

### 檢索效能指標
- **Recall@K**：不同截斷點（K=5,10,20）的召回率 ≥ 80%
- **Mean Reciprocal Rank (MRR)**：> 0.75
- **NDCG@10**：> 0.78（考慮排序品質）
- **延遲**：< 200ms（P95 延遲）

### 可解釋性指標
- **推理透明度分數**：> 90%（透過 SHAP-style 方法量化）
- **人工評估 I-R 對齐度**：> 85%（推理與檢索結果的一致性）
- **推理可完成度**：> 95%（能否為所有結果生成有據推理）

### 抗幻覺指標
- **幻覺檢測準確度**：> 92%（正確識別缺乏視覺證據的推論）
- **視覺證據覆蓋率**：> 98%（推論中提及的元素在投影片中的實際位置命中率）
- **最終用戶信任度**：> 4.5/5（用戶調查評分）

## 6. 風險與緩解策略

| 風險 | 影響 | 緩解策略 |
|------|------|---------|
| ColPali 向量提取性能瓶頸 | 批處理速度 | GPU 批處理優化、分佈式計算 |
| 多向量儲存成本過高 | 基礎設施成本 | 向量量化、內存映射文件、增量索引 |
| MM-R5 推理延遲 | 端到端性能 | 模型蒸餾、推理優化（ONNX/TensorRT）、異步處理 |
| 視覺幻覺難以完全消除 | 用戶信任 | 多層驗證、置信度動態調整、保守性輸出 |
| 跨域投影片泛化性 | 模型魚性 | 微調數據擴增、域適應技術 |
